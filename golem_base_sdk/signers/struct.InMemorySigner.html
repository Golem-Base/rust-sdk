Struct InMemorySigner

Source 
pub struct InMemorySigner { /* private fields */ }
Expand description

A signer that keeps the private key in memory and supports loading, saving, and listing keys. Useful for local development and testing, with support for keystore files and raw key files.

Source

pub fn get_keystore_dir() -> Result<PathBuf>

Gets the default keystore directory path according to the XDG spec. Creates the directory if it does not exist.

Source

pub fn generate() -> Self

Generates a new random private key and returns an in-memory signer. This is useful for creating new accounts programmatically.

Source

pub fn private_key(&self) -> SigningKey

Returns the private key as a SigningKey. Can be used for exporting or further cryptographic operations.

Source

pub fn public_key(&self) -> VerifyingKey

Returns the public key as a VerifyingKey. Useful for verifying signatures or exporting the public key.

Source

pub fn save(&self, password: &str) -> Result<PathBuf>

Saves the private key to a file in the standard keystore directory using the provided password. The file is encrypted and named after the account address.

Source

pub fn load_keystore(path: PathBuf, password: &str) -> Result<Self>

Loads a private key from a keystore file at the given path using the provided password. Returns an in-memory signer if successful.

Source

pub fn load_by_address(address: Address, password: &str) -> Result<Self>

Loads a signer by address from the default keystore directory. Looks for a file named after the address and decrypts it with the given password.

Source

pub fn load_raw_key(path: PathBuf) -> Result<Self>

Loads a signer from a raw private key file (not encrypted). Expects the file to contain the raw private key bytes.

Source

pub fn list_local_accounts() -> Result<Vec<Address>>

Lists all local accounts found in the keystore directory. Returns a vector of Address for each account found.
